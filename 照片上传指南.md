# 减脂记录 · 照片上传指南

每天往减脂记录（`content/posts/fat-loss-journey/index.md`）数据表里加进度照的流程。

> 一句话：**照片丢进 `inbox/` → 跑 `npm run photo` → 提交部署**。
> 脚本自动去 EXIF、缩图、传 R2、把缩略图写进表格对应日期那行。

---

## 一、一次性准备（已配好，换电脑才需要重做）

1. 装依赖（仓库根目录）：

   ```bash
   npm install
   ```

2. 复制 `.env.example` 为 `.env`，填 5 个值：

   | 变量 | 哪里拿 |
   |------|--------|
   | `R2_ACCOUNT_ID` | Cloudflare 账号 ID |
   | `R2_BUCKET` | R2 bucket 名 |
   | `R2_ACCESS_KEY_ID` / `R2_SECRET_ACCESS_KEY` | R2 → Manage R2 API Tokens 建的 S3 密钥（权限 Object Read & Write）|
   | `IMG_DOMAIN` | 交付域名，如 `https://cdn.neepoo.dev`（结尾不要带 `/`）|

   `.env` 已被 git 忽略，不会提交。

---

## 二、日常上传（每天就这几步）

### 1. 放照片到 `inbox/`

文件名 / 文件夹名 **必须是当天日期**。两种放法都行：

```
inbox/2026-05-26.jpg              ← 直接放文件，名字就是日期
inbox/2026-05-26/任意名字.jpg      ← 或放进日期命名的文件夹（里面图片名随意）
```

- 只放**正面 1 张**。文件夹里多张时，脚本只取第一张。
- 日期取自**文件名 / 文件夹名**，所以文件夹里那张图叫什么都行（之前 `2025` 那种手误也不影响）。
- 支持 jpg / png / heic / heif / webp，统一转成 jpg 上传。

### 2. 确保数据表已有那天的行

脚本只**填照片列**，不会新建行。所以先在 `index.md` 数据表里把当天的体重那行记好，例如：

```
| 2026-05-26 |      | 79.0      | 22.0       | 92.5      | 今天的训练备注 |
```

（照片列留空就行，脚本来填。）

### 3. 先预览（可选，不上传）

```bash
npm run photo:dry
```

只处理 + 改 md，**不上传 R2、不动原图**，用来确认它认出了日期、找到了表格行。

### 4. 正式跑

```bash
npm run photo
```

它会自动：

1. 烤入 EXIF 方向 → **去掉所有元数据（含 GPS 定位）** → 长边缩到 1600 → 转 JPEG
2. 上传到 R2，key = `fat-loss/2026-05-26.jpg`
3. 找到表格里 `2026-05-26` 那行，把照片列填成缩略图（点击开大图）
4. 把原图移到 `inbox/done/`（不进 git）

终端会打印每张的上传大小和「✓ 已写入表格行」。

### 5. 提交部署

```bash
git add content/ 照片上传指南.md
git commit -m "添加 2026-05-26 进度照"
git push
```

原图和 `.env` 已被 `.gitignore` 挡住，不会误传。

## 四、常见问题

| 现象 | 原因 / 处理 |
|------|-------------|
| 缩略图没出来 / 像裂了 | 多半是浏览器缓存了旧的失败请求 → **Ctrl+Shift+R 硬刷**。表格最后一行图是 `loading="lazy"`，**没滚到不加载**，往下滚一下即可 |
| 提示「表格里没有 XXXX 这行，跳过」 | 先在数据表加当天的数据行，再重跑 |
| 提示缺环境变量 | `.env` 没填全，对照第一节 |
| 想换缩略图大小 / 画质 | 改 `scripts/photo.mjs` 顶部的 `MAX_EDGE`（原图长边）、`THUMB_W`/`THUMB_H`（缩略图显示尺寸）、`quality` |

---

## 五、它是怎么显示的（技术说明）

- 原图只存一份在 R2（`fat-loss/日期.jpg`），缩放全靠 Cloudflare 的 `/cdn-cgi/image/` 实时变换。
- 缩略图 URL：`width=120,height=160,fit=cover,gravity=auto,format=auto,quality=80`
  - `format=auto` → 浏览器支持就发 AVIF/WebP，省流
  - `fit=cover,gravity=auto` → 裁成统一 3:4，自动对焦主体
  - 显示 60×80，源取 2x（120×160）保证视网膜清晰
- 点击大图 URL：`width=1080,format=auto,quality=82`
- 表格里 `<img>` 带 `max-width:none`，否则会被主题的 `img{max-width:100%}` 在窄列里压扁。
