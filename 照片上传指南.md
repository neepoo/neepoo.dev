# 减脂记录 · 照片上传指南

每天往减脂记录（`content/posts/fat-loss-journey/index.md`）数据表里加进度照，以及每月往训练月报里加统计截图的流程。

> 一句话：**照片丢进 `inbox/` → 跑 `npm run photo` → 提交部署**。
> 脚本自动去 EXIF、缩图、传 R2、把缩略图写进表格对应日期那行。
>
> 月报截图：**截图丢进 `inbox/YYYY-MM-summary/` → 跑 `npm run summary:images -- --month YYYY-MM` → 提交部署**。

---

## 一、一次性准备（已配好，换电脑才需要重做）

1. 装依赖（仓库根目录）：

   ```bash
   npm install
   ```

2. 复制 `.env.example` 为 `.env`，填 5 个值：

   | 变量 | 哪里拿 |
   |------|--------|
   | `R2_ACCOUNT_ID` | Cloudflare 账号 ID |
   | `R2_BUCKET` | R2 bucket 名 |
   | `R2_ACCESS_KEY_ID` / `R2_SECRET_ACCESS_KEY` | R2 → Manage R2 API Tokens 建的 S3 密钥（权限 Object Read & Write）|
   | `IMG_DOMAIN` | 交付域名，如 `https://cdn.neepoo.dev`（结尾不要带 `/`）|

   `.env` 已被 git 忽略，不会提交。

---

## 二、日常上传（每天就这几步）

### 1. 放照片到 `inbox/`

文件名 / 文件夹名 **必须是当天日期**。两种放法都行：

```
inbox/2026-05-26.jpg              ← 直接放文件，名字就是日期
inbox/2026-05-26/任意名字.jpg      ← 或放进日期命名的文件夹（里面图片名随意）
```

- 只放**正面 1 张**。文件夹里多张时，脚本只取第一张。
- 日期取自**文件名 / 文件夹名**，所以文件夹里那张图叫什么都行（之前 `2025` 那种手误也不影响）。
- 支持 jpg / png / heic / heif / webp，统一转成 jpg 上传。

### 2. 确保数据表已有那天的行

脚本只**填照片列**，不会新建行。所以先在 `index.md` 数据表里把当天的体重那行记好，例如：

```
| 2026-05-26 |      | 79.0      | 22.0       | 92.5      | 今天的训练备注 |
```

（照片列留空就行，脚本来填。）

### 3. 先预览（可选，不上传）

```bash
npm run photo:dry
```

只处理 + 改 md，**不上传 R2、不动原图**，用来确认它认出了日期、找到了表格行。

### 4. 正式跑

```bash
npm run photo
```

它会自动：

1. 烤入 EXIF 方向 → **去掉所有元数据（含 GPS 定位）** → 长边缩到 1600 → 转 JPEG
2. 上传到 R2，key = `fat-loss/2026-05-26.jpg`
3. 找到表格里 `2026-05-26` 那行，把照片列填成缩略图（点击开大图）
4. 把原图移到 `inbox/done/`（不进 git）

终端会打印每张的上传大小和「✓ 已写入表格行」。

### 5. 提交部署

```bash
git add content/ 照片上传指南.md
git commit -m "添加 2026-05-26 进度照"
git push
```

原图和 `.env` 已被 `.gitignore` 挡住，不会误传。

## 三、月度训练报告截图

这个流程用于 `content/posts/training-summary-YYYY-MM/index.md` 这种月报文章，把跑步、骑行、力量/器械的月统计截图上传到 R2，并插到对应小节的数据表后面。

### 1. 准备月报文章

先确保月报文章已经存在，例如：

```
content/posts/training-summary-2026-06/index.md
```

文章里需要有对应小节和数据表：

- `跑步`
- `骑行`
- `力量训练` / `椭圆机` / `健身房`

脚本会根据这些标题找到插图位置，并从小节里的表格自动生成图注。

### 2. 放截图到月份目录

推荐直接用固定文件名：

```
inbox/2026-06-summary/running.jpg
inbox/2026-06-summary/cycling.jpg
inbox/2026-06-summary/gym.jpg
```

如果手机导出的文件名是随机的，也可以不改名，但要用 `--order` 明确告诉脚本排序后的文件分别是什么。比如当前 2026-05 的三张随机截图，按文件名排序后是：

```bash
npm run summary:images -- --month 2026-05 --order gym,cycling,running
```

`--order` 的值只支持这三个类型：

| 类型 | 含义 |
|------|------|
| `running` | 跑步月统计 |
| `cycling` | 骑行月统计 |
| `gym` | 健身房、力量训练、椭圆机月统计 |

### 3. 先预览（可选）

```bash
npm run summary:images -- --month 2026-06 --dry-run
```

只检查图片、生成待上传内容和待写入文章的结果，**不上传 R2、不改 md**。

如果截图文件名是随机的，预览时也要带 `--order`：

```bash
npm run summary:images -- --month 2026-06 --order running,cycling,gym --dry-run
```

### 4. 正式跑

```bash
npm run summary:images -- --month 2026-06
```

它会自动：

1. 烤入 EXIF 方向 → 去元数据 → 缩到 1080 宽 → 转 JPEG
2. 上传到 R2，key = `training-summary/2026-06/running.jpg` 这类路径
3. 在月报文章里插入对应截图块
4. 如果之前插过同一个月份同一类型的截图，重跑会替换旧块，不会重复堆图

如果图片已经传过，只想重新写文章，可以用：

```bash
npm run summary:images -- --month 2026-06 --no-upload
```

## 五、常见问题

| 现象 | 原因 / 处理 |
|------|-------------|
| 缩略图没出来 / 像裂了 | 多半是浏览器缓存了旧的失败请求 → **Ctrl+Shift+R 硬刷**。表格最后一行图是 `loading="lazy"`，**没滚到不加载**，往下滚一下即可 |
| 提示「表格里没有 XXXX 这行，跳过」 | 先在数据表加当天的数据行，再重跑 |
| 月报提示 `Cannot infer activity` | 截图文件名无法判断是跑步/骑行/健身。把文件改名为 `running.jpg`、`cycling.jpg`、`gym.jpg`，或加 `--order` |
| 月报提示 `Cannot find heading` | 月报文章里没找到对应小节标题。确认有 `跑步`、`骑行`、`力量训练与椭圆机` 这类标题 |
| 提示缺环境变量 | `.env` 没填全，对照第一节 |
| 想换缩略图大小 / 画质 | 改 `scripts/photo.mjs` 顶部的 `MAX_EDGE`（原图长边）、`THUMB_W`/`THUMB_H`（缩略图显示尺寸）、`quality` |

## 六、它是怎么显示的（技术说明）

- 原图只存一份在 R2（`fat-loss/日期.jpg`），缩放全靠 Cloudflare 的 `/cdn-cgi/image/` 实时变换。
- 缩略图 URL：`width=120,height=160,fit=cover,gravity=auto,format=auto,quality=80`
  - `format=auto` → 浏览器支持就发 AVIF/WebP，省流
  - `fit=cover,gravity=auto` → 裁成统一 3:4，自动对焦主体
  - 显示 60×80，源取 2x（120×160）保证视网膜清晰
- 点击大图 URL：`width=1080,format=auto,quality=82`
- 表格里 `<img>` 带 `max-width:none`，否则会被主题的 `img{max-width:100%}` 在窄列里压扁。
- 月报统计截图存到 R2 的 `training-summary/YYYY-MM/` 下，文章里用 `monthly-summary-image:<type>` 注释包住，方便脚本下次重跑时精准替换。
